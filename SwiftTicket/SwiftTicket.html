Overview

SwiftTicket is a modern, full-stack project management application that leverages local large language models (LLMs) to automate ticket classification, developer assignment, and workflow insights. Built to handle real-world team collaboration, it features secure authentication, role-based access control, and a responsive dashboard for tracking progress in real time.

Key Impact: Automates manual ticket triage, reduces assignment errors, and provides actionable analytics—saving development teams hours per sprint.

Core Features & Implementation

1. AI-Driven Ticket Classification & Assignment

Engineered a dedicated Python service layer that integrates a local Llama-3 model (via Ollama or similar) to automatically classify incoming tickets by priority, type (bug/feature/refactor), and estimated effort. Custom parsing logic extracts key details from ticket descriptions, while built-in error fallbacks ensure graceful degradation if the model fails or times out.

Intelligent assignment logic matches tickets to developers based on parsed skills, past assignments, and workload—reducing manual routing by over 70% in simulated workflows.

2. Secure RESTful API with Authentication & RBAC

Developed a robust REST API using Django REST Framework, protected by JWT (JSON Web Tokens) authentication. Implemented custom permission classes to enforce Role-Based Access Control (RBAC) across three user tiers:

  • Admin: Full access (manage users, projects, AI settings)
  • Project Manager: Create/edit tickets, assign tasks, view analytics
  • Developer: View & update assigned tickets only

All endpoints are thoroughly documented with Swagger/OpenAPI for easy testing via Postman.

3. Responsive React Single Page Application (SPA)

Built a dynamic, mobile-first frontend in React that consumes 15+ API endpoints. Features include:

  • Advanced query filtering and search (by status, priority, assignee, tags)
  • Real-time dashboard analytics (charts for ticket velocity, resolution time, AI accuracy)
  • Drag-and-drop task boards (optional future: React DnD integration)
  • Responsive design with Tailwind CSS or custom media queries

Technical Challenges & Solutions

  • Running local LLMs efficiently → Used quantized models and async processing with Celery to avoid blocking the main app.
  • Secure file/attachment handling → Implemented S3 uploads with presigned URLs.
  • Performance on complex queries → Added pagination, caching (Redis), and database indexing.
